First steps

This library aims to manage states in general, but its power resides when using asynchronous ones.

This section should let you get your hands dirty with the library, we will be using axios in our examples.

We will be building an application for users management, but we will go incrementally, so you can have an overall idea on how the library is flexible and its way of state management.

In this section you will learn how to deal with the following using the library:

  • Manually trigger data fetch
  • Trigger data fetching based on dependencies
  • Search while typing (+ concurrency & debounce)
  • URL based automatic data fetching (via the query string)

The following examples are using useAsyncState hook.

You can think of its signature like

function useAsyncState(config, dependencies = []) {
  ...
}

Fetching the users listâ€‹

Trigger the fetch on button's clickâ€‹

The following snippet should get you started to the library:

  • Typescript
  • JavaScript
import axios from "axios";
import {
  useAsyncState,
  UseAsyncState,
  ProducerProps
} from "react-async-states";

const API = axios.create({
  baseURL: "https://jsonplaceholder.typicode.com"
});

type User = {
  id: number;
  email: string;
  name: string;
};

async function fetchUsers(props: ProducerProps<User[]>): Promise<User[]> {
  const controller = new AbortController();
  props.onAbort(() => controller.abort());

  const usersResponse = await API.get("/users", {signal: controller.signal});
  return usersResponse.data;
}

export default function App() {
  const {state, run}: UseAsyncState<User[]> = useAsyncState(fetchUsers);
  const {status, data} = state;

  return (
    <div className="App">
      <button onClick={() => run()}>Fetch users</button>
      <h3>Status is: {status}</h3>
      {status === "success" && (
        <ul>
          {data.map((user) => (
            <li key={user.id}>{user.name}</li>
          ))}
        </ul>
      )}
    </div>
  );
}

Make it automatic on mountâ€‹

Let's take a close look at how we used the useAsyncState hook in the previous example:

const {state, run}: UseAsyncState<User[]> = useAsyncState(fetchUsers);

This is equivalent to its base version:

const {state, run}: UseAsyncState<User[]> = useAsyncState({
  producer: fetchUsers,
});

The base way to useAsyncState hook is the use a configuration object. If sometimes your use case is basic, the library supported some shortcuts.

Fetching the user detailsâ€‹

React to dependencies with conditionâ€‹

Now, let's fetch the user details when typing his id.

This time, we will be:

  • Storing the userId in a state variable using react's useState.
  • Pass the userId to our producer in the payload.
  • Only fetch if the userId is truthy.
  • Fetch everytime the userId changes.
  • Abort the previous call if a second is done while pending.

Here is a full working example:

  • Typescript
  • JavaScript
async function fetchUser(props: ProducerProps<User>): Promise<User> {
  const controller = new AbortController();
  props.onAbort(() => controller.abort());

  const {userId} = props.payload;

  const usersResponse = await API.get("/users/" + userId, {
    signal: controller.signal
  });
  return usersResponse.data;
}

export default function App() {
  const [userId, setUserId] = React.useState("");
  const {state}: UseAsyncState<User> = useAsyncState(
    {
      lazy: false,
      condition: !!userId,
      producer: fetchUser,
      payload: {
        userId
      }
    },
    [userId]
  );
  const {status, data, props} = state;

  return (
    <div className="App">
      <input onChange={(e) => setUserId(e.target.value)}/>
      <h3>Status is: {status}</h3>
      {status === "success" && (
        <details open>
          <pre>{JSON.stringify(data, null, 4)}</pre>
        </details>
      )}
      {status === "error" && (
        <div>
          error while retrieving user with id: {props?.payload.userId}
          <pre>{data.toString()}</pre>
        </div>
      )}
    </div>
  );
}


Try it here, notice the cancellation of previous requests, also, you can remove the abort callback and/or the signal to make concurrency chaos, and make sure to observe the consistency in the UI.

Debounce search while typingâ€‹

If we look at the previous example, we run a fetch request whenever the user types something (and we ignore empty string).

Now, let's debounce the run by 400ms:

That's all we need to do it:

// ...
export default function App() {
  const [userId, setUserId] = React.useState("");
  const {state}: UseAsyncState<User> = useAsyncState(
    {
      lazy: false,
      condition: !!userId,
      producer: fetchUser,
      runEffect: "debounce",
      runEffectDurationMs: 400,
      payload: {
        userId
      }
    },
    [userId]
  );
  // ...
}

tip

The run function supports passing parameters to the producer, received as args.

Let's edit the previous example and get rid of the state variable:


async function fetchUser(props: ProducerProps<User>): Promise<User> {
  const controller = new AbortController();
  props.onAbort(() => controller.abort());

  const [userId] = props.args;

  const usersResponse = await API.get("/users/" + userId, {
    signal: controller.signal
  });
  return usersResponse.data;
}

export default function App() {
  const { run, state }: UseAsyncState<User> = useAsyncState({
    lazy: true,
    producer: fetchUser,
    runEffect: "debounce",
    runEffectDurationMs: 400
  });
  const { status, data, props } = state;

  return (
    <div className="App">
      <input onChange={(e) => run(e.target.value)} />
      <h3>Status is: {status}</h3>
      {status === "success" && (
        <details open>
          <pre>{JSON.stringify(data, null, 4)}</pre>
        </details>
      )}
      {status === "error" && (
        <div>
          error while retrieving user with id: {props?.payload.userId}
          <pre>{data.toString()}</pre>
        </div>
      )}
    </div>
  );
}

Notice that we removed the state variable and also the payload and the dependency array.

The library's default dependencies are an empty array.

React to URL changeâ€‹

Now, rather than writing the user id and reacting to a state variable, let's grab the userId from the url and navigate when the typed value changes.

// highlit-next-line
const {userId} = useParams();
const {state}: UseAsyncState<User> = useAsyncState(
  {
    lazy: false,
    condition: !!userId,
    producer: fetchUser,
    payload: {
      userId
    }
  },
  [userId]
);
const {status, data, props} = state;


// AND

const navigate = useNavigate();

function onChange(e) {
  const id = e.target.value;
  if (id) {
    navigate(`/users/${id}`);
  }
}
<input onChange={onChange} />;
